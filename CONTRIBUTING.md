# Contributing to Retflo™

Thanks for being here. Retflo is built on the idea that political reasoning can
be structured, shared, and made into a commons. Contributions — whether code,
argument content, corrections, or documentation — make that more true.

## Before you contribute

Retflo uses a Contributor License Agreement (CLA). You don't need to read the
whole thing before opening a PR — a bot will handle it automatically the first
time you contribute.

**What the CLA does:**
- You keep copyright of your work
- You grant the project the rights needed to distribute it under the RCCL and
  to issue commercial licenses to organizations that would otherwise be excluded
- Commercial licensing revenue goes toward sustaining the project and funding
  cooperative and community infrastructure

**What the CLA doesn't do:**
- It doesn't take ownership away from you
- It doesn't let the project be sold to or taken over by an investor-owned entity
  (the license and the CLA both prevent this)

When you open your first pull request, a CLA bot will comment with a one-click
sign process. It takes about 30 seconds.

## What to contribute

**Argument nodes and connections**
The graph currently covers 59 arguments across 7 domains. If you see a gap —
an objection that isn't addressed, a connection that isn't mapped, a response
that could be stronger — open an issue or submit directly.

If you're modifying existing argument content, mark your changes clearly and
explain your reasoning. The integrity of the argument structure matters.

**Code**
Bug fixes, API improvements, visualizer features, tooling, documentation — all
welcome. Check open issues for things already flagged.

**Translations and accessibility**
The argument graph in other languages would significantly expand reach. If you
have the skills and the context to translate accurately (not just linguistically
but politically), reach out or open an issue first to coordinate.

**Corrections**
If something is factually wrong, historically inaccurate, or structurally
misrepresents a position — open an issue. The framework is only as good as its
accuracy.

## How to contribute

1. Fork the repository
2. Create a branch for your change
3. Make your changes with clear commit messages
4. Open a pull request with a description of what you changed and why
5. Sign the CLA if it's your first contribution (the bot will prompt you)

For significant changes to argument content or structure, open an issue first
to discuss — it saves everyone time.

## Standards

- Represent objections in their strongest form. No strawmen.
- Cite sources for factual or historical claims where relevant.
- Keep the reasoning oriented toward the framework's stated purpose: human
  flourishing and cooperative governance.
- Be direct. The project doesn't benefit from diplomatic vagueness.
